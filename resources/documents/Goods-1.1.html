Goods Api - Version-1.1

1. 取得商品列表(/goods) get接口修改

新增 必填项 参数 status: status 表示商品状态, 共有

-2：等待审核，-1：审核未通过，0：下架，1：上架，2：删除

5种状态, 状态间可以联合使用

例子1: 取得等待审核商品 : url = /goods?shop_id=10061&offset=0&limit=10&status=-2

例子2: 取得等待审核和审核未通过商品 : url = /goods?shop_id=10061&offset=0&limit=10&status=-2,-1

2. 修改商品信息接口新增非充要参数 (/goods/${goods_id}) status

可以实现商品上架, 下架处理

例子1: 商品上架 url = /goods/75, method = put, body = {'status': 1}, return = {"is_success": true, "goods_id": 75}

例子2: 商品下架 url = /goods/75, method = put, body = {'status': 2}, return = {"is_success": true, "goods_id": 75}

3. 新增取得用户在某商店具体信息接口 (/customer/${user_id}/shop/${shod_id}/info), 提供 get() 方法

可以返回用户收藏商店的商品, 逛店次数, 逛店时间等信息
